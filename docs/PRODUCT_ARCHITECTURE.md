## 一、产品定义

### 1.1 这是什么

一个运行在 Codex Desktop 上的**中国法律 AI 智能体系统**。
输入自然语言的法律问题，输出符合中国法规范的结构化法律分析、文书和建议。

### 1.2 不是什么的

- 不是法律数据库（数据来自元典智库/北大法宝 MCP）
- 不是判决预测工具（JDP 是独立辅助模块）
- 不替代律师判断（所有输出均标注"须经执业律师审核"）

### 1.3 核心价值

```
传统律师工作                        本系统
─────────────────                  ─────────────────
手动检索法条                    →  语义树+法条全文自动匹配
凭经验判断请求权基础              →  请求权基础分析法强制推理
用美国法概念思考                  →  29项阻断清单实时过滤
类案检索费时费力                  →  MCP并行检索+结构化对比
文书格式不统一                    →  标准化输出模板
```

## 三、仓库职责

| 仓库 | 角色 | 类比 |
|:-----|:-----|:-----|
| **CN** (Claude-for-Legal-CN-to-Codex) | 应用层：技能+知识+推理引擎 | 操作系统 |
| **MCP** (Codex-Claude-legal-cn-mcp-hub) | 连接层：外部数据源接入 | 网卡驱动 |
| **JDP** (Codex-Legal-CN-Judgment-Predictor) | 辅助层：判决预测的独立工具 | 专业软件 |
| **ALN** (PRC-US-Legal-Semantic-Alignment-Framework) | 基础层：中美法律概念映射的上游 | 底层库 |

### 依赖关系

```
ALN ──(自动同步)──▶ CN ◀──(MCP调用)── MCP
                     │
                     └──(知识注入)──▶ JDP
```

- CN 不依赖 JDP（JDP 是可选辅助工具）
- CN 依赖 MCP（通过 Codex MCP 协议，非硬依赖）
- CN 依赖 ALN（通过 sync CI 自动同步）
- JDP 依赖 CN（运行时读取 CN 知识库文件）

## 四、用户交互流程

### 4.1 主入口

用户打开 Codex Desktop，通过 `@claude-legal-cn` 触发。

```
用户输入："审查这份供应商合同"
    ↓
claude-legal-cn 路由：关键词"合同审查" → commercial-legal/review
    ↓
Rule Runtime 强制加载：
  - reasoning-template-zh.md (民事→请求权基础)
  - blocking-list.md (过滤 consideration/discovery 等)
    ↓
领域技能执行：
  - 读取 contract-review-playbook.md
  - 加载 contract-review-checklist.md (51项)
    ↓
MCP 调用（按需）：
  - 元典智库：搜同类合同纠纷裁判规则
  - 国家法规库：核验引用的法条是否现行有效
    ↓
输出：
  - 结构化审查备忘录
  - Rule Runtime 自检报告
  - ⚠️ 须经执业律师审核
```

### 4.2 技能路由规则

| 用户意图 | 触发关键词 | 目标技能 |
|:-----|:-----|:-----|
| 合同审查 | 审查合同/看看这个协议 | commercial-legal/review |
| 劳动问题 | 解除/竞业/加班费/工伤 | employment-legal |
| 诉讼准备 | 起诉/证据/代理词/质证 | litigation-legal |
| 判决预测 | 预测/胜算/能赢吗 | → 调用 JDP |
| 地域口径 | 北京怎么看/各地差异 | → regional-practice-guide |
| 知识查询 | XX法第几条/民法典规定 | → knowledge-base + 语义树 |

## 五、技术架构

### 5.1 Runtime 栈

```
Codex Desktop
  ├── LLM (Claude via Codex)
  ├── MCP Protocol
  │   ├── chineselaw (元典智库)
  │   ├── pkulaw (北大法宝)
  │   ├── flk-npc (国家法规库)
  │   └── rmfyalk (案例库)
  └── File System
      ├── ~/.codex/skills/ (CN 技能部署)
      └── ~/.codex/config.toml (MCP 配置)
```

### 5.2 推理流水线

```
输入 → [1]事实要素抽取 → [2]知识注入 → [3]Rule Runtime拦截
→ [4]领域推理执行 → [5]MCP联动(按需) → [6]输出自检 → 输出+自检报告
```

**Rule Runtime 拦截清单：**
- 推理链锁定：加载 reasoning-template-zh.md
- 阻断过滤：扫描 blocking-list.md
- 境外法域拦截：禁止引用美/英/港法律

**领域推理方法：**
- 民事：请求权基础分析法
- 刑事：犯罪构成要件检视
- 行政：行政行为合法性审查

**输出自检维度：**
- 法条准确性 ✅/❌
- 阻断概念漏网 ✅/❌
- 推理链合规 ✅/❌
- 逻辑完整性 ✅/❌
