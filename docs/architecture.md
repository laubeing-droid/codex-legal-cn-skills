## 设计定位

不是简单的"中文翻译"——是一层一层把美国法律 AI 替换为中国法律体系的适配工程：

```
┌────────────────────────────────────────┐
│  ⑥ 监控层（可选）——                    │
│     GitHub Actions + 4 路 diff-tool    │
│     上游更新了？通知我，我自己决定      │
├────────────────────────────────────────┤
│  ⑤ 护栏层（本仓库新增）——              │
│     阻断清单 / 元规则 / 香港桥梁        │
│     → 防止 AI 输出美国法内容            │
├────────────────────────────────────────┤
│  ④ PRC-US 概念对齐（本仓库新增）——     │
│     12 领域中美法律概念映射             │
│     → 让 AI 知道"美国这个等于中国那个"  │
├────────────────────────────────────────┤
│  ③ 法条层（qulv 本仓库新增）——         │
│     22 部中国法律官方 PDF 全文          │
│     → 替代美国法引用                    │
├────────────────────────────────────────┤
│  ② 内容层（来源于上游 + 本仓库修改）——  │
│     150+ 子技能 + 12 CLAUDE.md         │
│     → 5 个美国子技能已重写为中国版      │
│     → 12 个 CLAUDE.md 已中文化          │
├────────────────────────────────────────┤
│  ① 安装部署层（本仓库重写）——          │
│     install/update 从本仓库直接部署     │
│     → 你改了什么，用户就装什么          │
└────────────────────────────────────────┘

                           ▲
                           │ 全部从本仓库 skills/ 部署
                           │
                    ~/.codex/skills/
```

## 适配规模

| 层 | 原始（Anthropic 美国法） | 适配后（本仓库） |
|:---|:-----------------------|:----------------|
| 法条 | U.S.C./各州法典 | 22 部中国法律 PDF |
| 子技能 | 150 个美国法子技能 | 4 个完全重写 + 1 个删除 |
| 工作流 | 英文 CLAUDE.md | 12 个全中文 |
| MCP 连接 | 美国工具链 | 12 个中国工具 |
| 概念映射 | 无 | 12 对中美概念对照 |
| 护栏 | 无 | 8 个阻断/元规则文件 |
| 语境 | 无 | 14 个中国法背景提示 |
| 部署 | 依赖上游 vendor | 本仓库独立部署 |

## 部署流程

```
install.ps1
  ├── 遍历 12 个法律领域 → 复制全部文件
  ├── 单独处理 solo-law-firm（8 科室嵌套结构）
  ├── 克隆 MCP Hub 仓库
  └── 验证 14 个技能入口
```

## 上游检测流程

```
diff-tool-zhou.ps1 [-Diff] [-Update]
  ├── 拉取 zhou210712 最新代码
  ├── 比对 24 个主文件（哈希）
  ├── 比对 150+ 子技能（哈希）
  ├── 输出：绿=未变 / 黄=已变更 / 紫=新增 / 红=缺失
  └── -Diff 显示行级 git diff
```
